Analyze the existing application.

Before implementing:

1. Audit current UI.
2. List major visual issues.
3. Create implementation plan.

Implementation priorities:

1. Layout hierarchy
2. Typography hierarchy
3. Card reduction
4. Table improvements
5. Spacing system
6. Color system

Constraints:

- Do not change functionality.
- Do not change business logic.
- Do not change API contracts.
- Do not change data structures.
- Do not remove user actions.

Work screen by screen.

After each screen:

- Explain changes.
- Provide before/after summary.

Information density is preferred over excessive whitespace.

This is a professional financial tool.

Do not redesign it into a marketing-style dashboard.

Keep data visible and accessible.
